Parámetros

En ES5, teníamos varias formas de tratar con funciones que necesitaban valores por defecto, argumentos indefinidos, y parámetros nombrados. Con ES6, podemos lograr todo esto y más utilizando una sintaxis más concisa.
Parámetros por Defecto
function addTwoNumbers(x, y) {
    x = x || 0;
    y = y || 0;
    return x + y;
}
En ES6, podemos simplemente proveer valores por defecto en un función:
 function addTwoNumbers(x=0, y=0) {
    return x + y;
}


Parámetros REST
En ES5, teníamos un número indefinido de argumentos tal como:
  
function logArguments() {
    for (var i=0; i < arguments.length; i++) {
        console.log(arguments[i]);
    }
}
Utilizando el parámetro REST, podemos pasar una cantidad indefinida de argumentos:
 function logArguments(...args) {
    for (let arg of args) {
        console.log(arg);
    }
}


Parámetros Nombrados
Uno de los patrones en ES5 para tratar parámetros nombrados era utilizar las opciones de objeto patrón, adoptado de jQuery:

  
function initializeCanvas(options) {
    var height = options.height || 600;
    var width  = options.width  || 400;
    var lineStroke = options.lineStroke || 'black';
}
Podemos lograr la misma funcionalidad utilizando destructuring como un parámetro formal de una función
 ffunction initializeCanvas(
    { height=600, width=400, lineStroke='black'}) {
        // Use variables height, width, lineStroke here
    }
